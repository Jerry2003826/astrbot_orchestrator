# Security Policy

## 当前安全原则

本项目的重构目标之一，是把所有高风险路径默认收敛为安全边界：

- 默认拒绝不安全条件表达式
- 默认拒绝路径穿越与危险文件名
- 默认拒绝自动回退到本地执行
- 默认拒绝非管理员触发副作用写入
- 默认拒绝不安全 MCP 地址

## 高风险能力

以下能力被视为高风险副作用：

- 代码执行
- 文件写入与持久化
- Skill 创建
- MCP 配置
- 插件安装与更新

这些能力应始终由服务端进行权限判定，不能信任模型输出。

## 报告问题

如果你发现以下类型的问题，请优先报告：

- 任意代码执行
- 路径穿越
- 非管理员写文件
- 沙盒边界失效
- SSRF / 内网访问
- 配置污染
